Game

Learn how to exploit the full potential of the game object.

All the available options

fps, ups, aps and ops

That's right, you can choose the frames per second (fps), the updates per second (ups), the sprite animations per second (aps) and finally the optimizations per second (ops). More information on aps and ops coming right up.

let mySpecialGame = new Game({fps: 30, ups: 30, aps: 5, ops: 2})

Don't forget to deconstruct D2PX!

optimize

A true or false option which toggles the optimization of the game. It makes the game run around 5 to 10 times faster. It is on by default.

let mySpecialGame = new Game({optimize: true})

onrender, onupdate, onplay, onanimate, onoptimize, onpause

Is called when the respective event happens. The value will the a function, therefore we call it a callback function.

let myGame = new Game({onplay: (e) => {console.log("The game has started")}})

canvas

Only use it if you want to make a custom canvas.

let myCustomCanvasGame = new Game({canvas: myHTMLCanvas})

All methods

update, render, animate, optimize

Call it if only you want to force its execution.

play, pause

play starts the game and pause stops the game from rendering, updating, animating and optimizing

myGame.play()
myGame.pause()

setScene, setGui

Set scene selects the scene that you want to update, render, animate, and optimize. Same goes for the setGui, but for the GUI object.

myGame.setScene(myScene)

playScene, pauseScene

pauseScene only stops the updating of the selected scene. playScene resumes it.

myGame.pauseScene()
myGame.playScene()
