What is D2PX?

D2PX is a game library specialized in making 2D pixelated games with efficiency and ease.

Structure

Its structure is relatively simple. Each D2PX game contains a Game object which renders a scene. The scene itself contains entities. The entseities may as well contain other entities, but they will always contain a bound object and a sprite object.

Relative positioning

All entities will hative pos relativeition to their parents during the initialization to simplify sub-component movements.

All the object types

This game engine is indeed object oriented. Therefore, it contains a large variety of objects all having a very important role in this code ecosystem.

Game

The game object is the object that will run your game. More precisely, it's the instance that will render and update the selected scene at a certain framerate.

Scene

The scene object is the object that will render and update your entities. It will also be the one holding the camera object.

Entity

The entity object contains its sub-entities along with its own sprite (aka graphics). Only entities can contain boudaries for rectangular collision detection.

Sprite

The Sprite object is the one that will render the graphics for your entity. It can only render rectangular shapes, text and images of your choice.

Bounds

The bounds object is responsible for the collision of your entity.

Vector

Everything related to position and size will be stored inside a Vector object.

Camera

This object is responsible for moving the players view.
